Flags misuse of the per-field companion annotations that travel with @ClassBuilder:
  • @Formattable must apply to a String or Optional<String> field - used to emit a @PrintFormat overload setter.
  • @Negate must apply to a primitive boolean field - emits an inverse setter pair.
  • @Collector must apply to a Collection, List, Set, Iterable, or Map field - emits varargs / Iterable bulk setters, with opt-in single-element addX / putX, clearX, and putXIfAbsent (maps only).
  • @BuildFlag(pattern = ...) is only meaningful on CharSequence or Optional<String> fields - the validator cannot check a regex against other types.
  • @BuildFlag(limit = N) is only meaningful on CharSequence, Collection, Map, object arrays, or Optional of String / Number - the validator has no notion of size on other types.

Applies-to errors are reported at ERROR severity because they lead to invalid code generation (the emitted setter would not compile) or silently ignored runtime validation.

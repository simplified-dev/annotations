# Marketplace Screenshot Checklist

Hand-tuned demo project for capturing JetBrains Marketplace screenshots of
the Simplified Annotations plugin. Not published. Not on the runtime
classpath of `:plugin` or `:library`.

## Setup

1. Build and launch the sandbox IDE with the plugin installed:
   ```bash
   ./gradlew :plugin:runIde
   ```
2. In the sandbox, **File > Open** -> select `demo/` (this folder).
   Trust the project. Wait for Gradle import + indexing to finish.
3. Recommended IDE polish before capturing:
   - **Darcula** theme (Settings > Appearance > Theme)
   - **JetBrains Mono** at 16pt (Settings > Editor > Font)
   - Hide the project tool window (Alt+1) when capturing the editor only
   - Hide breadcrumbs / sticky lines if they crowd the shot
   - Disable inlay hints that distract (Settings > Editor > Inlay Hints)
4. Marketplace specs: 1280x800 minimum, 1920x1200 maximum, PNG or JPEG,
   <= 10 screenshots per plugin. Use Win+Shift+S or ShareX for capture;
   crop with the IDE window centred.

## Shot list

Order them in this sequence on the marketplace listing - it tells the
pitch in order: builder generation -> rich setters -> autocomplete pays
off -> contract grammar -> caller-side enforcement -> resource paths.

### 1. `User.java` - the headline @ClassBuilder

- Open `User.java`. The gutter icon (the SA badge) sits on the
  `@ClassBuilder` annotation line. Hover to surface the "generated by
  ClassBuilder" tooltip if you want it in-frame.
- Keep the whole record on screen. Caption suggestion:
  *"@ClassBuilder generates a fluent builder with runtime validation"*.

### 2. `Order.java` - rich companion annotations

- Open `Order.java`. Highlight the diversity: `@Collector`, `@Negate`,
  `@Formattable`, `Optional`. Caption:
  *"Collections, booleans, formatted strings, and Optionals each get a
  tailored setter shape"*.

### 3. `OrderUsage.java` - autocomplete payoff

- Open `OrderUsage.java`. Place the caret immediately after the dot in
  `Order.builder().` (line 25, after `Order.builder()`).
- Press Ctrl+Space to force the completion popup. Wait for it to fully
  populate, then capture.
- Crop tightly around the popup + the source line. Caption:
  *"All synthesised setters surface in autocomplete before the first
  compile"*.

### 4. `MathUtils.java` - extended @XContract grammar

- Open `MathUtils.java`. The interesting line is the contract string
  itself - put the caret in the middle of one and let the syntax read.
- Optional: hover the `@XContract` annotation - the inferred-annotation
  provider surfaces the synthesised `@Contract` in the doc popup. Capture
  with the doc popup visible to show the bridge to JetBrains' built-in
  data-flow. Caption:
  *"@XContract supersedes @Contract with relational comparisons,
  instanceof, and grouped boolean operators"*.

### 5. `ContractCallerDemo.java` - caller-side warning

- Open `ContractCallerDemo.java`. Each `MathUtils.*` call is reddened
  because the inspection proves the contract violation from the literal
  arguments.
- Hover any reddened argument - the tooltip reads
  *"Argument always violates contract"*. Wait for it to render, then
  screenshot. Caption:
  *"Contract violations are flagged at the call site, not just at
  declaration"*.

### 6. `AssetLoader.java` - @ResourcePath inspection

- Open `AssetLoader.java`. `LOGO` is silent (the file exists), `MISSING`
  is reddened, `brokenBase` is reddened on the `base` attribute itself.
- Hover `MISSING` to surface the *"Resource not found"* tooltip. Capture
  with both `LOGO` and `MISSING` in frame so the contrast reads. Caption:
  *"@ResourcePath verifies referenced files exist in the project's
  resource roots"*.

### 7 (optional). Quick-fix popup on @ResourcePath usage

- In `AssetLoader.load`, the body forwards a `base = "icons"`-tagged
  parameter into `getResourceAsStream` without prepending the base.
- Place the caret on the argument, press Alt+Enter, screenshot the
  intention popup with *"Prepend "icons/" to argument"* highlighted.
  Caption:
  *"Quick-fix prepends the missing base when forwarding to a resource
  loader"*.

## Composition tips

- Use a **light** theme for one shot (the headline) to widen marketplace
  appeal - dark themes scan as "developer", light themes scan as
  "approachable". The remaining shots can stay dark.
- Crop each shot to **exactly the same width**; the marketplace gallery
  scales each thumbnail and inconsistent widths look amateurish.
- Avoid capturing the title bar / window chrome. The IDE has a built-in
  *Tools > Capture Screenshot of the IDE Frame* (in some EAPs) - if
  unavailable, manually crop in Snip & Sketch.
- Each tooltip needs ~2 seconds to materialise. ShareX's "delayed
  capture" (default Ctrl+PrtSc with a 5s timer) is the easiest way.

## Reverting

The fixture is wired into `settings.gradle.kts` only so IntelliJ
auto-imports it. To remove the demo entirely:

1. Delete this folder (`demo/`).
2. Drop `"demo"` from the `include(...)` line in `settings.gradle.kts`.

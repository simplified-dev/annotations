Warns at call sites whose literal arguments deterministically trigger a fail or throws clause in the target method's @XContract.

This inspection is complementary to XContract (which validates the contract syntax on the method's declaration). It walks every method call and constructor call, resolves the target, and checks each clause whose return is fail or throws: if the condition is provably true given the call's constant arguments, the call is flagged.

Examples

// Declaration
@XContract("(param1 < 0) -> fail; (param1 >= this.length()) -> fail")
char charAt(int index) { ... }

// Warned - literal -1 satisfies (param1 < 0)
s.charAt(-1);

// Not warned - argument is a runtime value
s.charAt(i);

What is checked

  • Null-literal arguments against == null / != null.
  • Integer literals against comparisons (<, >, ==, etc.).
  • Boolean literals against == and !=.
  • &&, ||, ! combinations of the above.
  • Both paramN and named-parameter references.

What is conservatively skipped

Clauses using this, field access (.length(), .size(), .empty()), instanceof, or non-literal arguments never produce a warning even when they would violate the contract at runtime.

Settings

  • Warn on 'fail' clause violations - toggles checks against clauses returning fail.
  • Warn on 'throws' clause violations - toggles checks against clauses returning throws TypeName.
  • Highlight for confirmed call-site violations - severity used when a clause is proven to fire. Default is "Server Problem" (generic error-or-warning) since this is a strong diagnostic.

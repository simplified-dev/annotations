Flags @ClassBuilder-annotated classes that are missing one or more of the three bootstrap methods the annotation processor expects to see on the class:
  • public static Builder builder()
  • public static Builder from(ClassName instance)
  • public Builder mutate()

These three methods are the only wiring required to expose the generated <ClassName>Builder class as a first-class part of the annotated type's public API. Method names and visibility are configurable via the @ClassBuilder attributes (builderMethodName, fromMethodName, toBuilderMethodName, access); the inspection honors those overrides.

A quick-fix inserts all missing bootstrap methods at once, each decorated with the matching @XContract so IntelliJ data-flow analysis understands their fresh-object return shapes.

Reported at ERROR severity because call sites that use ClassName.builder()... will fail to compile without these methods - the generated <ClassName>Builder file does not add them to the target class, only the bootstrap ones do.

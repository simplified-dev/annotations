Verifies that every @ResourcePath-annotated field, parameter, or method return resolves to a resource that exists in the project's content source roots (or in user-configured additional roots).

What is checked

  • Missing resource file - the resolved path does not correspond to an existing file on the classpath. Reported at the annotation site or the argument expression.
  • Invalid base directory - the optional base attribute points to a folder that does not exist. Reported on the attribute value itself, with its own configurable severity.

Supported value resolution

The inspection statically evaluates the annotated expression and compares every possible resolved path against the classpath. Recognised value sources:

  • Direct string literals ("config/settings.xml")
  • Binary concatenation ("config/" + name)
  • Final / static / enum fields
  • Local variables (including UAST declarations)
  • Method calls - the called method's body is recursively evaluated
  • Enum constructor argument bindings

Evaluation is bounded in both recursion depth and result-set size to stay responsive in large projects; ambiguous branches degrade gracefully to "skip this branch" rather than halt analysis.

Examples

// Field with implicit root
@ResourcePath
private String configFile = "config/settings.xml";

// Field with base - validates config/settings.xml
@ResourcePath(base = "config")
private String configFile = "settings.xml";

// Method parameter - every call site's argument is checked
void loadResource(@ResourcePath(base = "config") String fileName) { }
loadResource("settings.xml");            // validates config/settings.xml

// Method return - validated against every return expression
@ResourcePath(base = "config")
String defaultConfigName() { return "settings.xml"; }

// Enum constructor
public enum Files {
    SETTINGS("settings.xml");             // validates config/settings.xml

    Files(@ResourcePath(base = "config") String path) { ... }
}

Companion inspection

The paired Resource Path Base-Prefix Usage inspection catches the common runtime mistake of passing a @ResourcePath(base = "X") parameter raw into a resource-loading call without prepending the X/ prefix at runtime.

Settings

  • Highlight for invalid base directory - severity applied when the base attribute points to a folder that does not exist. Independent from the main inspection severity (which governs missing-file reports).
  • Additional resource root paths - project-relative or absolute paths searched in addition to the project's content source roots. Useful for generated resources (e.g. build/resources/main) or projects with non-standard layouts.
  • Excluded file patterns - glob patterns for files the inspection skips entirely (e.g. **/generated/**, *Test*.java). Supports ** (any sequence including /), * (any sequence excluding /), and ? (single character).

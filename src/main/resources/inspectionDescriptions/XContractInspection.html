Reports syntax errors and semantic mistakes in @XContract annotation values and attributes.

@XContract is a drop-in replacement for JetBrains' @Contract with a richer grammar. The plugin synthesises an equivalent @Contract for IntelliJ's data-flow analysis, so a single @XContract is enough - you do not need to declare both.

The annotation's value is a semicolon-separated list of clauses of the form (condition) -> return. Each clause describes what a method returns or throws given that its arguments satisfy the condition. The empty form () acts as an unconditional fallback.

Condition grammar

OrExpr      := AndExpr ('||' AndExpr)*
AndExpr     := Term ('&&' Term)*
Term        := '(' OrExpr ')'                      // grouping
             | Value 'instanceof' TypeName         // type check
             | Value CompOp (Value CompOp)* Value  // chained comparison
             | '!' Value                           // boolean negation
             | Value                               // bare boolean
CompOp      := '<' | '>' | '<=' | '>=' | '==' | '!='
Value       := ('param'N | Name | 'this') ('.' SpecialField)*
             | 'null' | 'true' | 'false' | integer
SpecialField:= 'size()' | 'length()' | 'length' | 'empty()'
TypeName    := Ident ('.' Ident)*

Return values

true, false, null, !null, fail, this, new, paramN, a named-parameter reference, an integer constant, or throws TypeName.

Attributes

  • value - the contract clauses described above.
  • pure - declares the method has no observable side effects.
  • mutates - comma-separated list of this, io, or paramN.

Examples

// Named parameter reference
@XContract("(index < 0) -> fail")
char charAt(int index) { ... }

// Logical OR
@XContract("(param1 == null || param2 == null) -> fail")
void requireBoth(Object a, Object b) { ... }

// Grouping
@XContract("((param1 == null) && (param2 > 0)) -> fail")
void reject(Object a, int b) { ... }

// Boolean constant
@XContract("(flag == false) -> fail")
void requireTrue(boolean flag) { ... }

// Pure + mutates
@XContract(value = "() -> this", mutates = "this")
StringBuilder append(String s) { ... }

// instanceof check
@XContract("(param1 instanceof Number) -> true; () -> false")
boolean isNumeric(Object value) { ... }

// Typed throws
@XContract("(param1 == null) -> throws IllegalArgumentException")
void requireNonNull(Object value) { ... }

// Chained range comparison
@XContract("(0 <= index < this.size()) -> !null")
Object at(int index) { ... }

// Named-parameter return
@XContract("(param1 == null) -> defaultValue; () -> param1")
<T> T orElse(T value, T defaultValue) { ... }

Settings

  • Warn when an override weakens the super method's contract - toggles the inheritance check.
  • Highlight for contract inheritance violations - severity used for pure-downgrade and mutates-superset warnings. Independent from the main inspection severity (which applies to syntax errors and undefined references).
